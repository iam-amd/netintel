# Model Card: NetIntel Churn Scorer

## Purpose

NetIntel estimates whether a synthetic ISP customer profile may be at risk of
churn. It is designed as a portfolio demo and learning project.

## Data

The dataset is generated by `generate_dataset.py`. It is synthetic and includes
ISP-style operational signals:

- plan type and monthly revenue
- customer tenure
- data usage
- support tickets
- late payments and payment delay
- days since last contact
- recent outages
- average optical Rx power
- auto-pay status
- referrals brought

No real customer names, phone numbers, addresses, MAC addresses, billing records,
or OLT data are included.

## Model

The model is Logistic Regression.

Reasons for this choice:

- explainable coefficients
- easy to export to JSON
- small enough to score in the browser
- good fit for a student portfolio demo

Numeric features are standardized. Categorical features are one-hot encoded. The
React app reproduces the same transformation using `model/model_artifact.json`.

## Intended Use

Good use:

- portfolio demonstration
- learning ML + product integration
- showing how churn scoring could fit an ISP dashboard

Bad use:

- real customer decisions
- billing or disconnection decisions
- claiming production ML accuracy

## Limitations

- Synthetic data cannot prove real-world performance.
- The model has no real customer outcome validation.
- The explanation is coefficient-based, not causal.
- Real deployment would need anonymized data, monitoring, retraining, and privacy review.

## Ethical Note

Retention scoring should help operators support customers, not punish them.
Any real version should be used for service improvement, complaint follow-up,
and fair retention workflows.
